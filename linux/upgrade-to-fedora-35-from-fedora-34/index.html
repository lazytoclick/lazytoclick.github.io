從 Fedora 34 升級到 Fedora 35

發表於 2021-11-04 更新於 2023-08-03 分類於 linux 文章字數： 566 所需閱讀時間 ≈ 1 分鐘

前言

昨天，我在 Parallels Desktop 15 的虛擬環境下去，安裝一個全新的 Fedora 35 Workstation。

接著，我把之前安裝在外接硬碟的 Fedora 34 拿出來更新，並且以我自己的使用習慣快速感受一下升級後的不同，以下是我的升級流程。

升級流程

  1. 我的 Fedora 34 安裝完之後，在今天升級到 Fedora 35 之前，完全都沒有更新過。sudo dnf upgrade –refresh 這道指令下完後，CLI(Terminal) 顯示這些要更新的套件，總共 1.4G，光在這裡，就花費一點時間 。
  2. 走完 sudo dnf system-upgrade download –releasever=35 這道指令時，已經快過 2 個小時。
    5. 更新系統相較於全新安裝，對我來說，有個好處是有些更新就相對顯而易見。
  3. 升級完不久，系統馬上再跳出更新套件的訊息。
  4. 更新系統，也可以透過 GUI 達成。這個經驗，姑且就留給下次吧！

後記

  1. 其實…，我對系統的要求沒有很多，已經有好幾年，無論是那一個系統的升級，我認為要重新適應的地方，通常不多。
  2. 安裝硬體：Mac mini(2014 年末)、1TB 外接硬碟。
  3. 更新系統參考資料網址：DNF System Upgrade :: Fedora Docs
