來玩 Linux Mint 21.2

發表於 2023-07-29 更新於 2023-12-24 分類於 linux 文章字數： 2.8k 所需閱讀時間 ≈ 5 分鐘

前言

不久前，我留下一篇 來一場 Linux Mint 21.1 與 Manjaro 22.1.0 之間的 PK 的紀錄，當時的 Linux Mint 21.1 在 HP ProDesk 600 上的表現，讓我感到有點意外。

然而，就在 2023 年 7 月 16 日，迎來 Linux Mint 21.2，號稱 Victoria 這個新版本(以下，我會把 Linux Mint 21.2 與 Victoria 這 2 個名稱混搭使用)。

儘管最近工作忙到，下班就是想耍廢，然而，久久未被點燃的環法熱火，今年被皮卡車與賣魚哥的纏鬥再次激發，索性就一邊看環法，一邊來玩玩看 Linux Mint 21.2。

快速回顧與感想

  1. 穩定：「相較於上回我自己在 Linux Mint 21.1 上的體驗，這次是我自己在近些日子來，重新嘗試不少不同的 Distro 的經驗基礎上，我發現 Linux Mint 21.2 表現是非常穩定的。」

    呃…，上面這一段基本上就是「複製貼上」的概念，但，我必須說，這一段說法，在往下走之前，我認為比較適用於「在 VirturalBox 上運行」的前提下，而今次，我正是想看看 Victoria 能不能夠「在 HP ProDesk 600 上運行」，也同樣有穩定的表現。

    直接說結論：穩定性確實有提升，但不能說是完美。

    Victoria 主要應付我文書處理、上網，看 Youtube 與 Netflix 等需求，期間發生過幾次桌面當機 (如下圖)，當機後其他程式仍然可以運行，但，相關的桌面行為，例如：螢幕截圖，就無法運作，所以，我只好用手機拍照。我發現，當機的主要原因是在 CPU 無法獲得良好的散熱導致。

  2. 好上手的桌面環境：一直以來，面對 Linux Mint，我都是搭配服用 Cinnamon Edition 的桌面環境，由於使用上很順手，也就沒有特別想到什麼。這一次，我花比較多時間與 Victoria 相處，當我把 Microsoft Edge 這支瀏覽器安裝進來之後，使用的感受上，似乎就更接近 Windows，不禁讓我想起 Linux Lite。然而，我必須說，對比之下，Linux Mint 給我感受更貼近 Windows。

    相信對於不少 Windows 想要轉用 Linux 的伙伴而言，是一個滿大的加分選項，我相信，這或許可以解釋 Linux Mint 何以能夠在 DistroWatch 「網頁參觀排名」上排在第 3 名。

  3. 輕快的使用體驗，以及我喜歡的系統預載程式：Victoria 延續 Linux Mint 21.1 帶來的「輕快」使用感受，雖然不一定是最快的一支 Distro，但，我相信有前段班的水準(純綷是我個人感受，並沒有實測數據)。另外，防火牆與輸入法的部分，分別都已預先安裝 gufw 與 Fcitx5(搭配酷音輸入法)，也一樣從上一版延續下來，我個人是很喜歡的。

中文輸入法框架的使用心得

  1. 推薦的輸入法框架：我想，預設的 Fcitx5 就是最佳的選擇，如果是新酷音輸入法的使用者，基本上系統安裝完成，就可以直接使用。

  2. 嘸蝦米輸入法的話，要費點功夫，但也不難處理，我把說明放在下文。

安裝嘸蝦米輸入法

  1. 關鍵是 fcitx5-table-extra 這支套件，但身為 ubuntu(Debian) 衍生發行版的 Linux Mint，也一樣沒有將 fcitx5-table-extra 納入軟體庫。

  2. 我的方法是先下一道指令，一口氣把能夠上的 Fcitx5 相關套件都補足。

  3. 跑一趟 pkgs.org (Fcitx5-table-extra Download (RPM, TGZ, TXZ, XBPS, XZ, ZST)) 把 fcitx5-table-extra 載回來。

  4. 載回來的是一個壓縮檔：fcitx5-table-extra-5.0.13-1-any.pkg.tar.xz，我建議不要急著解壓縮以及把檔案放到它們該去的位置：/usr/share 裡，因為，「重新啟動」Fcitx5 後，會發現嘸蝦米的「嘸」圖像，並沒有正常顯示，而新酷音的圖像卻是大大地展露無遺。(詳可見下圖)

  5. 基本上，這個的狀況，不太會影響到嘸蝦米輸入法的使用(至少我使用過程沒遇到過)。但，我多少有點在意，因此走一個簡單的做法，就是只把 fcitx5-table-extra-5.0.13-1-any.pkg.tar.xz裡 /usr/share/icons/hicolor/16x16/apps 這個路徑底下的嘸蝦米圖像相關檔案(詳可見下圖)原封不動地搬入系統內的 /usr/share/icons/hicolor/16x16/apps 裡，其他 icons/hicolor 路徑內的圖像檔，如：/22x22、/24x24、/48x48 以及 64x64 底下的圖像檔，就不要移師進入系統 (按兵不動的概念)。

  6. 接著，「重新啟動」Fcitx5，應該會顯示 16x16 的嘸蝦米圖像(詳可見下圖)，雖然，在 48x48 的新酷音圖像面前，嘸蝦米顯得有點小咖就是，但，總比沒有好。當然，如果要讓嘸蝦米與新酷音平起平坐，也不是不行，但…，對我來說，有顯示就有加分，我已經懶得再花時間去處理這美中不足之處。

  7. 以上說明，比較不是以實際的步驟，一個接著一個帶出來，若不夠詳盡，或許可以再參考之前的這篇貼文：在 ubuntu 22.04.1 LTS 上用 Fcitx5 搭配嘸蝦米 (feat. fcitx5-table-extra)。

後記

  1. 最後，我想說一下，一個比較不負責任的推測，就是…，寫下 來一場 Linux Mint 21.1 與 Manjaro 22.1.0 之間的 PK 那一篇時，Linux Mint 的表現確實讓我感到意外，但在新版本：Victoria 有比較明顯的改善，然而，在 HP ProDesk 600 上面，還是會因為 CPU 發熱而影響到系統運作，雖不致於全然掛點，但，發熱後，系統明顯受到影響仍然會發生，我這才意識到 HP ProDesk 600 是一部小主機，Linux Mint 並不像 Manjaro 自行開發相對應的硬體設備。

    湊巧的是 Manjaro 開發不少小主機，這或許可以說明，Manjaro 為何能夠在 HP ProDesk 600 有比較好的表現。

    當然，我也相信 Linux Mint 21.2 在一般的主機上，肯定會有更好的發揮。

  2. 本次安裝使用的 ISO 檔是 linuxmint-21.2-cinnamon-64bit.iso。
